# <LG CNS 6기] 3일차 TIL — 개발환경(IDE·Live Server)과 HTML 태그·CSS 선택자

> TL;DR: 웹 개발환경(IDE·서버·Live Server)을 잡고 HTML 태그 구조·속성·CSS 선택자 기초까지. 과거 SAP S/4HANA 공부 때 쓰던 이클립스가 사실 'IDE 그 자체'였다는 걸 이제야 개념으로 정리함.

## 오늘의 학습 키워드
- IDE(이클립스·vscode), 웹서버 vs WAS, Live Server(로컬 실행)
- HTML 태그·중첩, 속성(attribute), 대소문자 구분
- CSS 선택자, Material Icon Theme

## 공부한 내용 (내 언어로 정리)

### 5강 — 웹 서비스의 이해 (개발환경 구성)
웹개발을 하려면 먼저 개발 도구인 IDE가 필요함. 현업에서 가장 주로 쓰는 건 이클립스고, vscode도 많이 씀.

웹을 열려면 서버가 필요한데, 서버에는 웹서버 또는 웹애플리케이션서버(WAS)가 있음. 다만 지금 단계에서 정식 서버까지 띄우는 건 무거우니까, 간단하게 **Live Server**로 html 파일을 바로 열어서 확인함. html은 태그 형식으로 구성됨.

에디터 환경은 Material Icon Theme 같은 걸 깔면 파일 아이콘이 세련되게 바뀌어서 보기 편해짐.

### 6강 — HTML 기초
- html은 **대소문자 구분**이 중요함.
- html은 태그로 이루어져 있고, 태그는 **중첩**이 가능함(태그 안에 태그).
- 태그는 **속성(attribute)** 을 가짐 — 태그에 붙는 부가 정보.
- Live Server로 열면 로컬(local)에서 웹이 뜨는데, 아직 css·script가 없으면 **디자인이 없는 뼈대만** 나옴.
- css에서는 **선택자**로 "어떤 태그에 스타일을 걸지"를 지정함.

정리하면 오늘로 Step 1(웹 서비스 이해) 6강을 마무리함. 개념(요청·응답·API)에서 이제 실제 화면을 만드는 HTML·CSS로 넘어온 셈.

## 트러블슈팅 (막힌 지점 · 해결 과정)
오늘도 실습 에러보다 "예전에 잘못 알고 있던 개념"을 바로잡는 게 핵심이었음.

- **IDE가 뭔지 몰랐던 과거의 오해 → 이번에 정리.**
  - 문제: 예전에 SAP S/4HANA를 공부하며 이클립스를 썼는데, 그때는 IDE가 뭔지도 몰라서 "요즘 뜨는 cloud 기술이 IDE인가 보다" 정도로 뭉뚱그려 생각했음.
  - 원인: 도구(개발 환경)와 언어(개발 대상)를 구분 못 했던 것. 이클립스 안에서 ABAP을 쓰던 걸 하나로 묶어서 이해함.
  - 해결: 이번 강의로 정리됨 — **이클립스 자체가 IDE(개발 도구)** 이고, 그 안에서 **ABAP이라는 언어로 개발**을 하는 구조였음. 도구≠언어. cloud와도 무관.
- **웹서버랑 WAS 차이?** → 어제 배운 WS vs WAS와 연결됨. 웹서버는 완성된 정적 파일(html·css·js)을 그대로 전달하고, WAS는 그 위에서 동적 로직·데이터 처리를 담당함. 오늘은 그걸 정식으로 안 띄우고 Live Server로 로컬 확인만 함.
- **Live Server로 열었는데 왜 디자인이 없지?** → html 뼈대만 있고 css·script를 아직 안 붙였기 때문. css를 선택자로 태그에 걸기 시작하면 그때부터 디자인이 입혀짐.

## AI 활용 기록 (썼을 때만)
- 오늘은 기초 개념 위주라 AI에 기대지 않고 강의를 직접 따라가며 정리함. 오히려 과거 교양 웹프로그래밍 때 "어차피 안 쓸 것"이라며 AI로 결과물만 뽑았던 습관을 반성하는 계기가 됐음 — 지금은 기초를 내 손으로 다시 밟는 중.
