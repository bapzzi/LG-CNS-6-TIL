# <LG CNS 6기] 9일차 TIL — 변수와 타입(기본 vs 참조)·메소드 4유형·클래스를 만드는 이유

> TL;DR: Java 변수를 본격적으로 다뤘다. (1) **변수는 데이터를 담는 그릇**이고, 변수타입은 **기본타입**(값을 담음)과 **참조타입**(주소를 담음) 둘로 나뉜다. 값을 담는 건 숫자·문자·논리형이고, 그 외는 전부 참조타입이라 **기본타입부터 단단히** 외워야 한다(정수 기본 `int`, 실수 기본 `double`). (2) 클래스 구성요소는 **변수 + 메소드**다. 메소드는 `[접근지정자] [반환타입] [메소드이름(매개변수)] {}` 형식이고, **매개변수·반환 유무 조합으로 4유형**이 나온다(반환 없으면 `void`). (3) 클래스를 만드는 이유는 추상적인 요소를 **프로그래밍 영역으로 끌고 와 instance를 만들기** 위해서다.

## 오늘의 학습 키워드

**변수와 타입**

| 용어 | 내 정리 |
|------|---------|
| **변수** (variable) | 데이터를 담을 수 있는 그릇 |
| **기본타입** (primitive) | 데이터(값) 자체를 담는 타입 |
| **참조타입** (reference) | 값이 아니라 **주소**를 담는 타입. 기본타입을 제외한 모든 타입 |
| **접근지정자** | `public` / `private` (이번엔 존재만 확인, 세부는 나중) |
| **전역변수 / 지역변수** | 선언 위치에 따른 구분. 오늘은 **클래스 내부** 기준으로만 다룸 |

**메소드**

| 용어 | 내 정리 |
|------|---------|
| **메소드** (method) | 클래스의 동작. 클래스 구성요소는 변수 + 메소드 |
| **반환타입** | 메소드가 돌려주는 값의 타입. 안 돌려주면 `void` |
| **매개변수** (parameter) | 메소드가 받는 입력 |

## 공부한 내용 (내 언어로 정리)

### 1. 변수는 그릇, 타입은 "값이냐 주소냐"

변수는 한마디로 **데이터를 담는 그릇**이다. 여기서 한 겹 들어가면 변수타입이 나오는데, 타입은 크게 두 갈래다.

- **기본타입**: 그릇에 **값 자체**를 담는다.
- **참조타입**: 그릇에 값이 아니라 **주소**를 담는다.

값을 직접 담을 수 있는 건 숫자·문자·논리형이고, 이 값 타입들을 제외한 나머지는 전부 참조타입에 해당한다. 참조타입은 종류가 많아서, 지금 단계에서는 **기본타입부터 개념을 단단히** 잡고 가는 게 낫다고 정리했다. "값을 담는 소수의 타입"과 "주소를 담는 나머지 전부"로 경계를 그어두는 셈이다.

### 2. 기본타입 9가지 — 정수 기본은 int, 실수 기본은 double

값을 담는 기본타입은 아래 9가지다. **이 9가지를 제외하면 전부 참조타입**이라, 이건 확실히 외워둬야 뒤가 편하다.

| 분류 | 타입 |
|------|------|
| 숫자형(정수) | `byte`, `short`, `int`, `long` |
| 숫자형(실수) | `float`, `double` |
| 문자형 | `char` |
| 논리형 | `boolean` |
| 문자열 | `String` ※ |

> ※ 강의에서는 값 타입 묶음으로 `String`까지 함께 "9가지"로 짚어줬는데, 엄밀히 말하면 **`String`은 참조타입**이다(기본타입은 위 8개). 자주 쓰고 값처럼 다뤄져서 같이 외우게 짚어준 것으로 이해했다 — 아래 트러블슈팅에 따로 정리했다.

숫자형 정수가 `byte`·`short`·`int`·`long`으로 4개나 나뉘는 이유는 **효율성** 때문이다. 담을 숫자의 크기에 맞춰 골라 쓰라는 것인데, 다 외우려 하기보다 **정수는 `int`가 기본, 실수는 `double`이 기본**이라고 잡아두면 된다.

### 3. 변수 구문 형식과 선언 위치

변수 선언에는 정해진 구문 형식이 있다.

```
[접근지정자] [변수타입] [변수명] = value ;
```

그리고 변수는 **선언 위치**에 따라 전역변수와 지역변수로 나뉜다. 오늘은 이걸 **클래스 내부 기준**으로만 봤다 — 클래스 안에서 어디에 선언됐느냐로 전역/지역을 가른 것이다.

이 대목은 ABAP과 비교하며 들었다. ABAP에서도 전역/지역변수를 공부했는데, 거기서는 **프로그램 단위**에서 변수를 선언하고 진행했다. Java는 OOP라서 그런지 **클래스 단위**로 잡히는 게 좀 다르게 느껴졌다. 오히려 예전에 이클립스에서 해봤던 cloud 서버 쪽 OOP와 비슷한 감각이었다.

### 4. 클래스 규칙 재확인 + 변수 선언 실습

새 클래스를 만들 때 규칙은 어제와 같다.

- **파일명 = 클래스명**
- **클래스명은 대문자로 시작**

접근지정자는 아직 자세히 다루지 않았고, `public`과 `private`가 있다는 것만 알아두기로 했다.

오늘 실습은 자바의 기본적인 **변수 선언**을 해보는 것이었다. 다만 이 클래스에는 **`main` 메소드가 없어서 실행이 불가능**하다는 특징이 있었다. 여기서 클래스의 구성요소가 정리됐다 — 클래스는 **변수와 메소드**로 이뤄진다. 실행하려면 진입점인 `main` 메소드가 있어야 한다는 걸, 실행이 안 되는 클래스를 직접 보며 확인한 셈이다.

지역변수 선언은 이런 모양이다.

```java
public static void main(String[] args) {
    // 지역변수
    String msg = "이 곳은 프로그램의 시작점 입니다.";
}
```

### 5. 메소드 — 구문 형식과 4유형

메소드의 구문 형식은 이렇다.

```
[접근지정자] [반환타입] [메소드이름(매개변수)] { }
```

메소드는 **반환타입 유무**와 **매개변수 유무**의 조합으로 4가지 유형이 나온다.

| | 반환 있음 | 반환 없음 (`void`) |
|---|---|---|
| **매개변수 있음** | 입력받고 결과를 돌려줌 | 입력받지만 안 돌려줌 |
| **매개변수 없음** | 입력 없이 결과만 돌려줌 | 입력도 반환도 없음 |

반환이 없으면 반환타입 자리에 **`void`**를 쓴다. 매개변수와 반환에는 각각 원하는 타입을 지정할 수 있다.

이 틀로 어제부터 계속 보던 `main`을 다시 읽으면 해석이 된다.

```java
public static void main(String[] args)
```

- 반환타입이 `void` → **아무것도 반환하지 않는다.**
- 매개변수가 `String[] args` → **매개변수로 String(배열)을 받는다.**

늘 쓰던 `main` 한 줄이 "반환 없음 + String 매개변수 받음"이라는 걸, 유형 분류를 배우고 나서야 문장처럼 읽혔다.

### 6. 클래스를 만드는 이유

정리하는 질문이 하나 나왔다 — **왜 클래스를 만드는가.** 답은, 현실의 추상적인 무형·유형 요소를 **프로그래밍 영역으로 끌고 오기** 위해서다. 그리고 그렇게 만든 클래스는 결국 **instance를 만들어내기 위해** 쓴다. class는 설계도, instance는 그 설계도로 실제 찍어낸 것 — 어제 잡은 구분이 "왜 굳이 class냐"라는 질문과 이어졌다.

## 트러블슈팅 (개념 정정 — String은 기본타입인가 참조타입인가)

오늘도 실행 에러는 없었다. 대신 개념에서 정정한 지점이 **String의 분류**였다.

- **문제**: 값을 담는 기본타입을 외우면서 `String`을 그 9가지 안에 같이 넣어 기억했다.
- **원인**: 강의가 값 타입 묶음으로 `String`까지 함께 짚어줬고, `String`이 값처럼 자주 쓰여서 기본타입으로 묶어 외운 것이다. 하지만 기본타입의 정의는 "값 자체를 담는 타입"인데, `String`은 그 정의에 정확히 들어맞지 않는다.
- **해결**: 확인해 보니 **기본타입은 8개**(`byte`·`short`·`int`·`long`·`float`·`double`·`char`·`boolean`)이고, **`String`은 참조타입**이다. 즉 `String` 변수는 문자열 값을 직접 담는 게 아니라 그 문자열이 있는 **주소를 담는다**. 오늘 배운 "기본=값 / 참조=주소" 기준을 그대로 적용하면 String이 참조 쪽이라는 게 앞뒤가 맞았다.

기억용으로는 강의처럼 "String 포함 9가지"로 묶어두되, **정의상 기본은 8개·String은 참조**라는 걸 겹쳐서 잡아뒀다.

## 추가로 찾아본 내용 (영상 밖 · 보충)

> 아래는 강의 범위를 넘어, "값을 담음 / 주소를 담음"이 실제로 어떻게 갈리는지 궁금해서 따로 확인한 것.

기본타입과 참조타입이 값/주소로 갈린다는 건, 저장되는 위치가 다르다는 뜻이기도 하다. 기본타입은 변수 자리에 **값이 바로** 들어가고, 참조타입은 변수 자리에는 **주소만** 있고 실제 데이터(예: 문자열 내용)는 다른 곳에 따로 있다. 그래서 참조타입은 "이 변수가 가리키는 대상"이라는 한 단계가 더 있는 것이고, `String`이 참조타입이라는 것도 이 그림으로 이해하니 자연스러웠다. (스택/힙 같은 정확한 용어까지는 오늘 범위가 아니라, "값을 직접 담느냐 / 주소로 가리키느냐"의 차이로만 잡아뒀다.)

## AI 활용 기록
- 물어본 것: `String`이 기본타입인지 참조타입인지(강의에서 값 타입으로 묶여 헷갈려서) / 값 타입과 참조 타입이 저장되는 방식이 어떻게 다른지.
- 검증: 오늘 배운 "기본=값 / 참조=주소" 기준에 `String`을 대입해, 문자열은 값을 직접 담기보다 주소로 가리킨다는 쪽이 정의에 맞음을 확인. 기본타입 개수도 8개(String 제외)로 맞춰봤다.
- 내 판단: 외울 때는 강의처럼 묶어도, **정의는 "기본 8개 = 값 / String 포함 나머지 = 참조(주소)"**로 잡는 게 안 헷갈린다.

## 오늘의 회고
- 몰입도: 보통~높음. 변수를 "그릇"으로, 타입을 "값이냐 주소냐"로 잡으니 기본/참조 구분이 한 번에 정리됐다. 메소드 4유형도 표로 놓고 보니 늘 쓰던 `main` 한 줄(`void` + `String[] args`)이 문장처럼 읽혀서 남았다.
- ABAP과의 비교: 전역/지역변수를 ABAP은 프로그램 단위로, Java는 클래스 단위로 잡는 차이가 눈에 띄었다. OOP라 변수의 소속 범위 자체가 클래스로 내려온다는 감을 잡았다.
