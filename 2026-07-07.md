## 공부한 내용 (내 언어로 정리)

### 3강
웹 개발은 먼저 프로그래밍 언어를 정해야 함. 제일 핫한 건 자바, 그다음 파이썬 정도인데 결이 다름. 자바는 상용화되는 웹에 주로 쓰이고, 파이썬은 데이터 관련으로 많이 씀. 언어를 정하면 개발 도구를 고르는데 이걸 ide(integrated development environment)라고 함. nginx나 tomcat이 유명함.

웹은 결국 request랑 response로 이루어지고 이걸 체화하는 게 중요함. 데이터 다루는 기본이 crud(create·read·update·delete)임. 요청 명세는 get, post, put, patch, delete 같은 메소드로 하고 이게 api임.

사용자 요청은 body랑 header로 나뉨. header에는 본문 타입이나 인증 토큰 같은 걸 담는데 메타데이터로 보면 됨. 요청 url은 명사 복수형으로 쓰고 계층 구조를 반영하는 게 좋은 설계임(예: /users/{id}). 데이터는 json 형식으로 서버에 전달하는데, react로 만들면 header랑 body로 요청이 나가고 json으로 전달됨. 물론 요청 형식이 json만 있는 건 아님.

http는 데이터가 암호화가 안 돼서 해킹 위험이 있고, 이걸 보완한 게 https(s=secure)임. 응답은 200, 201 같은 상태코드랑 응답 header로 이루어지고, 상태코드는 정해진 룰임.

### 4강 (이어서)
fe는 브라우저 도구로 사용자에게 보여지는 부분을 구성하고 html·css·javascript로 만듦. html은 뼈대, css는 디자인, javascript는 로직 담당. jsp·asp도 html 기반인데 jsp는 자바 기반이라 동적 로직이 붙음.

css의 반응형 디자인은 media query로 하고, 이걸 편하게 해주는 게 bootstrap임. 창을 줄일 때 스타일이 바뀌는 게 반응형 템플릿.

spa는 react로 만들면 html이 아니라 jsx 파일로 나옴. 기존 html이랑 다르게 스크립트 기반이라 템플릿이 다름. spa는 한 번 로드된 html·css·js를 기반으로 동작하고, 페이지 이동할 때 필요한 데이터만 불러와서 화면을 갱신함. 그래서 빠르고 서버 부하가 줄지만 초기 로딩은 느림.

react는 라이브러리고 ui에 특화됨. 여기서 라이브러리랑 프레임워크 구분을 제대로 잡아야 하는데, 제어권에 차이가 있음. 코드 묶음만 있으면 라이브러리, 애플리케이션 흐름까지 포함하면 프레임워크임(제어의 역전). vue는 프레임워크 성격, react는 라이브러리 성격이고 둘 다 ui 특화임.

## 트러블슈팅 (헷갈린 개념 · 정리)
오늘도 실습 에러가 아니라 개념이 안 붙는 게 문제라, 궁금한 걸 하나씩 잡아봄.

- **vscode도 ide인가?** → 엄밀히는 코드 에디터로 분류되지만 확장을 깔면 ide처럼 씀. intellij·eclipse 같은 정통 ide는 컴파일러·디버거·빌드가 처음부터 통합돼 있고, vscode는 가벼운 에디터에 확장을 붙이는 구조. 기능상으론 ide 역할이 맞음.
- **crud랑 db의 dml 차이?** → 같은 걸 다른 층에서 부르는 거였음. crud는 개념(앱·api 층)이고, dml(insert·select·update·delete)은 그 개념을 sql로 실제 실행하는 언어(db 층)임. create=insert, read=select로 대응됨. crud=무엇을, dml=sql로 어떻게.
- **json이 정확히 뭐지?** → 계속 들어봤는데 이번에 잡음. javascript object notation의 줄임말인데, 이름에 java가 있어도 프로그래밍 언어가 아니라 데이터 형식임. key-value로 된 텍스트라 사람이 읽기 쉽고, 언어 안 가리고 서버랑 클라이언트가 데이터 주고받을 때 씀.
- **bootstrap이랑 클로드코드에서 본 bootstrapping이 같은 건가?** → 이름만 같고 다른 거였음. 강의의 bootstrap은 트위터가 만든 css 프레임워크(반응형·미리 만든 디자인 부품). 클로드코드의 bootstrapping은 "최소 상태에서 스스로 시작·초기 골격을 세팅한다"는 일반 용어. "스스로 일으켜 세운다"는 비유만 같고 도구로선 관계없음.
- **spa·스크립트 기반·component가 뭔지?** → spa는 html을 한 번만 로드하고 이동할 때 필요한 데이터만 받아 화면 일부만 바꾸는 방식. 옛날 웹은 서버가 완성된 html을 통째로 보냈다면, spa는 브라우저에서 javascript가 돌면서 화면을 그때그때 만들어서 "스크립트 기반"이라고 하는 거였음. component는 버튼·카드 같은 재사용 가능한 화면 조각(레고 블록)이고, react는 이걸 쌓아서 ui를 만듦.
